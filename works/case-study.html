Mobile App Redesign 2024

사용자 이탈률 40% 감소를 위한 온보딩 UX 혁신

복잡한 가입 절차로 인한 사용자 이탈 문제를 해결하고, 직관적인 온보딩 경험을 설계한 프로젝트입니다.

Role Lead UX Designer
Duration 3 Months
Tools Figma, Maze, Hotjar
01

Project Overview

프로젝트 배경

핀테크 스타트업 A사는 출시 6개월 만에 MAU 50만을 달성했지만, 온보딩 단계에서 67%의 사용자가 이탈하는 심각한 문제에 직면했습니다. 경쟁사 대비 2배 높은 이탈률은 비즈니스 성장의 주요 병목이었습니다.

타겟 사용자

20-35세의 디지털 네이티브 세대로, 간편하고 빠른 서비스를 선호합니다. 복잡한 절차에 대한 인내심이 낮고, 3분 이내에 핵심 기능을 경험하지 못하면 앱을 삭제하는 경향이 있습니다.

프로젝트 목표

  • 온보딩 완료율 67% → 85% 달성
  • 가입 소요 시간 5분 → 2분 이내 단축
  • 첫 주 리텐션 30% → 45% 향상

나의 역할

UX Research UI Design Prototyping Usability Testing
02

Problem Definition

데이터와 사용자 인터뷰를 통해 핵심 문제를 정의했습니다.

과도한 정보 요청

가입 시 12개 항목의 정보를 요구하며, 사용자는 "왜 이 정보가 필요한지" 이해하지 못했습니다.

73% 3단계에서 이탈

진행 상태 불명확

사용자가 현재 몇 단계에 있는지, 얼마나 남았는지 파악할 수 없어 불안감을 느꼈습니다.

4.2분 평균 이탈 시점

에러 처리 부재

입력 오류 시 구체적인 안내 없이 "다시 시도하세요"만 표시되어 사용자 좌절감이 컸습니다.

58% 에러 후 이탈

"가입하는 데 왜 이렇게 많은 정보를 요구하는지 모르겠어요. 그냥 빨리 앱을 써보고 싶은데..."

— 사용자 인터뷰 참가자 (28세, 직장인)

Design Principles

01

Progressive Disclosure

필요한 순간에 필요한 정보만 요청하여 인지 부하 최소화

02

Immediate Value

가입 완료 전에도 앱의 핵심 가치를 미리 경험할 수 있도록 설계

03

Trust Building

정보 요청 이유를 명확히 설명하여 신뢰 구축

04

Solution

핵심 문제를 해결하는 3가지 디자인 솔루션을 도출했습니다.

Solution 01

스텝 프로그레스 인디케이터

현재 진행 상황과 남은 단계를 시각적으로 명확하게 표시합니다. 각 단계별 예상 소요 시간을 함께 보여주어 사용자의 불안감을 해소합니다.

  • 동적 프로그레스 바 (완료율 실시간 반영)
  • 단계별 예상 시간 표시 ("약 30초 소요")
  • 건너뛰기 가능한 선택 항목 구분
Solution 02

점진적 정보 수집

12개 항목을 4개 필수 + 8개 선택으로 재구성했습니다. 핵심 기능 사용 후 자연스럽게 추가 정보를 요청하는 방식으로 전환했습니다.

  • 필수 정보 12개 → 4개로 축소
  • "나중에 입력" 옵션 제공
  • 정보 요청 이유 툴팁 추가
Solution 03

인라인 밸리데이션 & 가이드

실시간 입력 검증과 구체적인 오류 메시지로 사용자가 문제를 즉시 인지하고 수정할 수 있도록 했습니다.

  • 실시간 입력 검증 (타이핑 중 피드백)
  • 구체적 오류 메시지 ("@ 뒤에 도메인을 입력하세요")
  • 성공 상태 시각적 피드백 (체크 아이콘)

06

Reflection

What I Learned

"적을수록 좋다"는 단순한 원칙이 아니라, 사용자에게 가치를 증명한 후 정보를 요청하는 것이 핵심이었습니다. 데이터 기반 의사결정의 중요성과, 가설을 빠르게 검증하는 린 UX 방법론의 효과를 체감했습니다.

Challenges

초기 접근 시 사용자 인터뷰보다 데이터 분석에 치중했던 점이 아쉽습니다. 숫자만으로는 "왜" 이탈하는지를 파악하기 어려웠고, 인터뷰를 통해서야 핵심 인사이트를 얻을 수 있었습니다.

Next Steps

향후 프로젝트에서는 정량 + 정성 데이터를 동시에 수집하는 리서치 설계를 적용할 예정입니다. 또한 A/B 테스트를 더 세분화하여 어떤 요소가 가장 큰 영향을 미쳤는지 분석하고자 합니다.
